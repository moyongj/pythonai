# 全栈教育平台开发技能规范

## 一、核心能力概述

具备独立完成类似"码上成长"教育平台的全栈开发能力，能够从需求分析到部署上线完整交付，支持快速迭代功能模块（如将 Python 课程替换为 SQL Server 课程）。

---

## 二、关键技能模块

### 1. 网站规划与需求分析
- 基于业务目标进行功能拆解，输出完整需求文档和原型设计
- 使用流程图、用例图梳理用户交互流程（学习流程、管理员审核流程）
- 理解教育类产品领域知识，区分学习者、教师、管理员三类角色需求

### 2. 技术栈选型
- **前端**: Next.js 16+ (App Router)、React 19、TypeScript 5、Tailwind CSS 4、shadcn/ui
- **后端**: Next.js API Routes、Node.js、Server Components
- **数据库**: SQLite/PostgreSQL、Prisma ORM
- **认证**: Session-based Authentication、JWT
- **部署**: Vercel/Cloudflare Pages、CI/CD 配置

### 3. 架构设计
- 设计前后端分离架构，定义 RESTful API 接口规范
- 组件化开发，合理划分公共组件（UI库、布局组件、业务组件）
- 实现权限控制架构，区分普通用户与管理员路由
- 设计可扩展数据模型，支持课程内容灵活替换

### 4. 前端开发
- 响应式页面开发，实现现代化 UI（首页、登录、资源库、练习、聊天、评估、统计、管理后台）
- 复杂表单交互、状态管理、数据可视化（图表展示）
- 集成 shadcn/ui 组件库，定制主题样式
- 处理客户端与服务端渲染边界问题，优化 Hydration 性能

### 5. 后端开发
- 开发 RESTful API，实现用户认证、资源管理、代码执行、数据统计
- 设计中间件处理权限校验、请求日志
- 集成外部服务（代码执行引擎、AI聊天接口）

### 6. 数据库设计与实现
- 设计规范化数据表结构（用户表、资源表、题目表、评估表、会话表）
- 熟练使用 Prisma ORM 进行数据操作，编写高效查询
- 实现数据迁移脚本，支持测试数据生成

### 7. 部署与维护
- 配置 Next.js 生产构建，优化打包体积
- 部署至 Vercel/Cloudflare，配置环境变量
- 编写自动化脚本（构建、数据生成、文档生成）
- 监控应用运行状态，排查常见问题

## 三、实践经验要求

| 经验项 | 量化标准 |
|--------|----------|
| 项目交付 | 独立完成至少 2 个类似规模全栈项目上线 |
| 页面开发 | 3 天内完成单个复杂页面（含前后端） |
| 功能迭代 | 1 周内完成课程内容整体替换（如 Python → SQL Server） |
| Bug 修复 | 平均修复时间 < 4 小时 |
| 代码质量 | TypeScript 严格模式零类型错误，ESLint 检查通过 |

---

## 四、课程内容替换流程示例

**场景**: 将 Python 课程替换为 SQL Server 课程

**执行步骤**:
1. **数据层**: 更新 `src/lib/db.ts` 数据模型，调整题目类型、资源分类
2. **内容层**: 运行数据生成脚本，导入 SQL Server 课程数据
3. **前端层**: 更新资源页面、练习页面中的内容展示逻辑
4. **后端层**: 调整题目接口等业务规则
5. **验证**: 测试所有页面功能，确保 UI 一致、内容正确
