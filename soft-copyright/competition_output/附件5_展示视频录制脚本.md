### 第3章：功能演示 - AI代码智能评价（核心！）（1:35 - 3:35）

**⚠️ 这是评审标准中"功能完整性与交互体验"的关键得分点，务必详细展示**

**画面**：录屏操作，实际运行平台

**步骤3.1 - 登录（1:35 - 1:50）**
```
操作：输入学生账号 01 / 密码 123456 → 点击登录
旁白："首先使用学生账号登录平台..."
```

**步骤3.2 - 进入代码评价页面（1:50 - 2:05）**
```
操作：点击导航栏「代码评价」
旁白："进入核心的AI代码智能评价模块"
```

**步骤3.3 - 输入题目和代码（2:05 - 2:40）**
```
操作：
  1. 在「题目标题」框输入："计算1到100的和"
  2. 在「题目内容」区域输入完整的题目描述
  3. 在代码编辑器中输入一段包含典型错误的示例代码，例如：
     ```python
     # 故意留一个常见错误供AI检测
     s = 0
     for i in range(101):
         s = s + i     # 缩进正确但逻辑有误(应为range(1,101))
     print(s)
     ```
旁白："现在模拟一位学生提交一段包含典型问题的Python代码..."
```

**步骤3.4 - 提交评价并等待结果（2:40 - 3:00）**
```
操作：点击「开始评价」按钮 → 等待AI返回（10-30秒）
旁白："点击开始评价，系统调用AI大模型对代码进行分析..."
（此时可以加速播放等待过程，或保留真实时间以体现真实性）
```

**步骤3.5 - 详细讲解评价报告（3:00 - 3:35）** ⭐ 最重要部分
```
操作：用鼠标/红框依次高亮以下区域：

① 四维得分条形图（理解30/逻辑25/可读性25/语法20）
旁白："AI从四个维度给出了评分——题目理解与实现、逻辑思路、
    代码可读性和语法掌握，每个维度都有独立分数"

② 总体评语区域
旁白："这是AI生成的总体评语，指出了代码的主要问题和改进方向..."

③ 易错知识点匹配区域
旁白："特别值得一提的是，AI自动匹配到了具体的易错知识点——
    这里识别出了'循环边界条件'这一常见错误类型，
    这得益于我们构建的166项易错知识点库"

④ 提示建议区域

⑤ 推荐类似练习题

## 三、录制技术要求清单

| 项目 | 要求 | 推荐方案 |
|------|------|---------|
| 录屏工具 | 不限 | OBS Studio（免费）/ Camtasia / 剪映电脑版 |
| 视频格式 | MP4 | 导出时选择H.264编码 |
| 分辨率 | ≥720P | 推荐 1920×1080 (1080P) |
| 时长 | ≤8分钟 | 本脚本约7分30秒 |
| 文件大小 | ≤1200MB | 1080P下一般<500MB，足够 |
| 音频 | 清晰无杂音 | 用麦克风录制旁白，或后期配音 |
| 字幕 | 建议添加 | 方便评审观看（剪映一键生成字幕） |
| 文件名 | 固定格式 | XX学校_XX姓名_码上成长Python代码智能评价与学情诊断AI智能体.mp4 |

## 四、录制前准备清单

- [ ] 平台服务已启动并可访问（localhost:5000 或公网地址）
- [ ] 准备好测试账号（01/123456 学生，myj/123456 管理员）
- [ ] 准备一段含典型错误的示例代码用于评价演示
- [ ] 安装配置好录屏软件（OBS/Camtasia/剪映）
- [ ] 准备好麦克风或提前录制好旁白音频
- [ ] PPT制作完成（痛点分析页、数据对比页、创新总结页）
- [ ] 预演至少1遍，确认总时长不超过8分钟

## 五、注意事项

1. **真实性第一**：所有功能演示必须使用真实运行的系统，不能使用静态截图拼凑
2. **重点突出**：代码评价模块是最核心的创新点，应占用最长演示时间
3. **数据一致**：视频中的所有数据和附件3说明书中的数据必须完全一致
4. **语速适中**：正常说话语速，不要过快，确保评委听清每个要点
5. **画面清晰**：录屏时字体调大（浏览器缩放125%-150%），确保评审看清内容
6. **避免敏感信息**：如出现真实学生姓名/头像，请打码处理
